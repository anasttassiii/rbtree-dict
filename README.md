# Red-Black Tree and Dictionary

## Описание

Реализация **красно-черного дерева** (Red-Black Tree) и **ассоциативного массива** (словаря) на языке C.

Красно-чёрное дерево — это самобалансирующееся двоичное дерево поиска, в котором баланс осуществляется на основе "цвета" узла дерева, который принимает только два значения: "красный" и "чёрный". Это гарантирует, что все операции (поиск, вставка, удаление) выполняются за время **O(log N)**.

В проекте используются:
- Классическая реализация красно-черного дерева
- Фиктивный лист (`nil`) для упрощения кода
- Вращения (левое и правое) для балансировки
- Ассоциативный массив (словарь) поверх красно-черного дерева
- Автоматическая балансировка при вставке и удалении

## Тесты

Проект содержит автоматические тесты для всех структур данных:

**Тест красно-черного дерева:**
- Вставка 13 ключей
- Проверка обхода (inorder traversal)
- Поиск всех элементов
- Удаление 3 элементов
- Проверка размера после удаления

**Тест ассоциативного массива (словаря):**
- Вставка пар (ключ → значение)
- Обновление значения по ключу
- Получение значений по ключу
- Проверка наличия ключа
- Удаление пары по ключу
- Проверка размера и пустоты

**Тест реального примера:**
- База данных студентов (ID → оценка)
- Поиск студента по ID
- Удаление студента
- Вывод всех студентов по порядку



## Свойства красно-черного дерева

1. **Каждый узел** — красный или черный
2. **Корень** — всегда черный
3. **Листья (`nil`)** — всегда черные
4. **У красного узла** оба ребенка черные (нет двух красных подряд)
5. **Все пути** от узла до листьев содержат одинаковое количество черных узлов

## Операции и их сложность

| Операция | Сложность |
|----------|-----------|
| Поиск | O(log N) |
| Вставка | O(log N) |
| Удаление | O(log N) |
| Обход (inorder) | O(N) |

## Вращения

Для балансировки дерева используются два типа вращений:

**Левый поворот:**
x y
/ \ /
A y → x C
/ \ /
B C A B


**Правый поворот:**
x y
/ \ /
y C → A x
/ \ /
A B B C


## Случаи балансировки

### При вставке (3 случая):
1. **Дядя красный** → перекрашивание
2. **Дядя черный, узел - правый ребенок** → поворот влево + перекрашивание + поворот вправо
3. **Дядя черный, узел - левый ребенок** → перекрашивание + поворот вправо

### При удалении (4 случая):
1. **Брат красный** → перекрашивание + поворот
2. **Брат черный, оба его ребенка черные** → перекрашивание брата
3. **Брат черный, левый ребенок красный** → перекрашивание + поворот
4. **Брат черный, правый ребенок красный** → перекрашивание + поворот
